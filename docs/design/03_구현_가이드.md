## Phase 1: 데이터 적재 파이프라인

### 작업 순서

#### 1-1. 환경변수 설정 (`app/config.py`)
- `.env`에서 모든 접속정보 로드
- Settings 클래스로 관리
- 기본값 설정 (로컬 개발용)

#### 1-2. CSV → MariaDB (`load_csv_to_mariadb.py`)
- pandas로 CSV 읽기
- SQLAlchemy로 테이블 생성 + INSERT
- 인덱스 생성 (filename, doi, coverdate, FULLTEXT)
- **검증**: 전체 행 수 확인

#### 1-3. 텍스트 청킹 서비스
- **기본**: `app/services/chunker.py` — 고정 크기 문자 분할
- **고급**: `app/services/chunker_v2.py` — 5가지 전략 (recursive, sentence, semantic, section)
- chunk_size=512, chunk_overlap=50
- chunk_id, chunk_total_counts 부여
- **추천**: `section` 전략 (논문 섹션 구조 인식, 문장 절단 0건)
- 상세: `docs/design/06_고급_청킹_설계서.md`

#### 1-4. MariaDB → Milvus (`load_mariadb_to_milvus.py`)
- MariaDB에서 전체 논문 조회
- 청킹 → 임베딩 → Milvus INSERT
- 컬렉션 스키마: Dense(IVF_FLAT/IP) + Sparse(BM25)
- **검증**: 컬렉션 건수 확인, 샘플 검색 테스트

## Phase 2: Core 클라이언트 구축

### 작업 순서

#### 2-1. LLM Client (`app/core/llm.py`)
- OpenAI-compatible API 호출 (httpx)
- chat_completion (비스트리밍)
- chat_completion_stream (SSE 스트리밍)
- 토큰 사용량 추출 (usage_out dict)
- Langfuse @observe 데코레이터

#### 2-2. Embedding Client (`app/core/embeddings.py`)
- embed_texts (배치), embed_query (단일)
- OpenAI-compatible /v1/embeddings 호출

#### 2-3. Milvus Client (`app/core/vectorstore.py`)
- hybrid_search (Dense + Sparse + RRF)
- vector_search (Dense only)

#### 2-4. MariaDB Client (`app/core/database.py`)
- get_paper_by_id, get_papers_by_doi

- search_papers (keyword, author, dates)

#### 2-5. 서버 시간 도구 (`app/core/tools.py`)

#### 2-6. Langfuse 설정 (`app/core/langfuse_client.py`)
- Langfuse 초기화
- @observe 데코레이터 설정

## 구현 시 핵심 주의사항

### 반드시 지켜야 할 것
1. **모든 접속정보는 .env** — 코드에 하드코딩 절대 금지
2. **chunk_id=1 필터** — 편수 카운트는 반드시 MariaDB에서 chunk_id=1만
3. **OpenAI-compatible API** — LLM/Embedding 모두 동일 형식
4. **토큰 사용량 누적** — 덮어쓰지 않고 _accumulate_usage로 합산
5. **서버 시간 기준** — 상대 날짜는 반드시 get_current_datetime() 사용
6. **스트리밍 모드 분리** — stream_mode일 때 LLM 호출 스킵, messages만 저장
7. **1 질문 = 1 trace** — Langfuse 트레이싱 보장

### 흔한 실수
- Milvus에서 편수 카운트 → chunk 중복으로 잘못됨 → MariaDB 사용
- 토큰 사용량 덮어쓰기 → 중간 LLM 호출 사용량 손실
- 상대 날짜를 시스템 시간으로 계산 → 서버 시간 도구 사용
- 스트리밍에서 trace가 여러 개 생김 → @observe 범위 주의
