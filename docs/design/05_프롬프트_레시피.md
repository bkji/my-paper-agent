# Claude Code 프롬프트 레시피

> 이 문서는 Claude Code에게 각 Phase별로 어떤 프롬프트를 주면 되는지 모아놓은 레시피입니다.
> 각 프롬프트는 독립적으로 사용 가능하며, 순서대로 실행하면 전체 시스템이 구축됩니다.

---

## 사전 준비

프로젝트 시작 전, 아래 파일들을 준비:
1. `CLAUDE.md` — 04_CLAUDE_MD_템플릿.md 기반으로 작성
2. `data/sample_paper.csv` — 원본 데이터
3. `.env` — 환경변수 설정
4. `docs/design/` — 설계 문서 (01, 02)

---

## 프롬프트 1: 프로젝트 초기화 + 데이터 적재

```
data/sample_paper.csv 파일을 분석해줘. 컬럼 구조와 데이터 특성을 파악한 후:

1. requirements.txt 생성 (fastapi, uvicorn, langgraph, pymilvus, sqlalchemy, pymysql, httpx, python-dotenv, pydantic, langfuse, python-dateutil, sse-starlette, pandas)
2. .env.example 생성 (MariaDB, Milvus, LLM, Embedding, Langfuse, 서버 설정)
3. app/config.py — Settings 클래스 (.env에서 환경변수 로드, 기본값 포함)
4. load_csv_to_mariadb.py — CSV를 MariaDB에 적재 (테이블 생성 + INSERT + 인덱스)
5. app/services/chunker.py — 기본 텍스트 청킹 (고정 크기)
6. app/services/chunker_v2.py — 고급 청킹 (recursive, sentence, semantic, section 전략)
   - docs/design/06_고급_청킹_설계서.md 참조
7. load_mariadb_to_milvus.py — MariaDB에서 읽어서 청킹→임베딩→Milvus 적재
   - Dense: bge-m3 (EMBEDDING_DIM), IVF_FLAT/IP
   - Sparse: BM25 built-in function

실행하여 검증까지 해줘.
```

## 프롬프트 2: Core 클라이언트

```
docs/design/02_상세기능_설계서.md 의 "6. 외부 시스템 클라이언트 상세" 섹션을 참조하여 app/core/ 아래 클라이언트들을 구현해줘:

1. app/core/langfuse_client.py — Langfuse 초기화 + @observe 데코레이터
2. app/core/llm.py — LLM Client
   - chat_completion(messages, temperature, max_tokens, usage_out) → str
   - chat_completion_stream(messages, ...) → AsyncGenerator[str]
   - 모든 호출에 @observe 적용
3. app/core/embeddings.py — Embedding Client
   - embed_texts(texts) → list[list[float]]
   - embed_query(query) → list[float]
4. app/core/vectorstore.py — Milvus Client
   - hybrid_search(query_vector, query_text, expr, top_k) → Dense+Sparse+RRF
   - vector_search, bm25_search (개별 검색)
5. app/core/database.py — MariaDB Client
   - get_paper_by_id, get_papers_by_doi, get_paper_fulltext_by_doi
   - search_papers(keyword, author, dates)
   - aggregate_papers(coverdate_from/to, keyword, author, group_by, volume, issue) — chunk_id=1만!
   - list_papers(...)
6. app/core/tools.py — 서버 시간
   - get_current_datetime() → dict
   - get_current_date_context() → 한국어 문자열
```

## 프롬프트 3: 에이전트 프레임워크

```
docs/design/02_상세기능_설계서.md를 참조하여 에이전트 프레임워크를 구축해줘:

1. app/agents/state.py — AgentState TypedDict
   - query, user_id, filters, messages, search_results, context, answer, sources, trace_id, metadata

2. app/agents/common.py — 공통 헬퍼
   - retrieve_by_query(query, user_id, filters, top_k) — Milvus hybrid search
   - multi_query_retrieve(queries, user_id, filters, top_k_per_query) — 다중 쿼리 + 중복 제거
   - llm_json_call(system_prompt, user_prompt, state, temperature) — JSON 파싱
   - llm_text_call(system_prompt, user_prompt, state, temperature) — 텍스트 (stream_mode 지원)
   - inject_date_context(system_prompt, state) — 날짜 + 대화 히스토리 주입
   - format_context(search_results) — 검색 결과 → LLM 컨텍스트
   - build_sources(search_results) — 검색 결과 → SourceDocument
   - extract_paper_title_from_history(state) — "이 논문", "N번째 논문" 참조 해결
   - _accumulate_usage(state, usage_out) — 토큰 누적 (덮어쓰지 않음)

3. app/core/date_parser.py — 한국어 날짜 파서
   - D1: 절대 특정 ("2024년 11월", "2024년 10월~12월")
   - D2: 분기/반기/연도 범위 ("2023년 3분기", "2022~2024년")
   - D3: 상대 ("최근 6개월", "작년", "올해", "작년 여름")
   - D4: 비교 ("2020년과 2024년 비교")
   - 서버 시간 기준, regex 기반

4. app/agents/supervisor.py — Supervisor (LangGraph StateGraph)
   - build_history → extract_dates → extract_conditions → classify_intent → route_to_agent → append_citation
   - INTENT_SYSTEM_PROMPT로 14개 에이전트 유형 분류
```

## 프롬프트 6: API 레이어

```
docs/design/02_상세기능_설계서.md의 "2. API 상세 설계" 섹션을 참조하여 API를 구현해줘:

1. app/models/schemas.py
   - ChatRequest(query, agent_type?, user_id?, filters?, messages?, stream?)
   - ChatResponse(answer, sources?, trace_id?, usage) ← usage는 필수(항상 반환)
   - SourceDocument(paper_id, title, author?, doi?, chunk_id, chunk_text, score)
   - UsageInfo(prompt_tokens, completion_tokens, total_tokens)
   - extract_usage() 헬퍼: 결과에서 usage 추출 (항상 유효한 dict 반환)

2. app/models/db_models.py — SQLAlchemy Paper 모델

3. app/api/deps.py — Bearer 토큰 인증 (OPENAI_COMPAT_API_KEY 빈값이면 비활성화)

4. app/api/routes/chat.py — POST /api/chat
   - 비스트리밍: supervisor.ainvoke → ChatResponse
   - 스트리밍: supervisor(stream_mode) → SSE (token/sources/usage/done 이벤트)
   - 1 질문 = 1 Langfuse trace (스트리밍에서도!)

5. app/api/routes/openai_compat.py
   - GET /v1/models → co-scientist 모델
   - POST /v1/chat/completions → messages 변환 → supervisor → OpenAI 형식 응답
   - X-OpenWebUI-User-Name 헤더 → user_id

6. app/api/routes/agents.py — GET /api/agents (14개 목록)
7. app/api/routes/documents.py — ingest + search

8. app/main.py — FastAPI 앱 + 라우터 등록 + CORS

서버 실행하여 /v1/models, /api/agents, /api/chat 동작 확인해줘.
```

## 디버깅 프롬프트 모음

### 날짜 파싱 문제
```
date_parser.py에서 "{실패한 질문}"을 파싱하지 못해.
date_parse_testcases 테이블의 테스트를 모두 통과하도록 수정해줘.
```

### Intent 분류 오류
```
"{질문}"이 {잘못된 에이전트}로 분류되는데 {올바른 에이전트}여야 해.
INTENT_SYSTEM_PROMPT를 개선해줘.
```

### 토큰 사용량 누락
```
API 응답에 usage가 0으로 나와.
common.py의 _set_final_usage와 llm.py의 usage_out 추적을 점검해줘.
참고: API에는 최종 답변 생성 호출만 반환. 내부 호출은 Langfuse에서 확인.
```

### 스트리밍 trace 중복
```
Langfuse에서 스트리밍 요청마다 trace가 여러 개 생겨.
chat.py의 @observe 범위를 확인하고 1질문=1trace로 수정해줘.
```

### Milvus 검색 결과 없음
```
Milvus 검색이 빈 결과를 반환해.
1. 컬렉션에 데이터가 있는지 확인
2. 임베딩 차원이 EMBEDDING_DIM과 일치하는지 확인
3. 필터 expr이 올바른지 확인
```
