# Co-Scientist Agent 설계 문서

> 이 문서 세트만 있으면 `data/sample_paper.csv` 1개로 전체 시스템을 Claude Code로 재구축할 수 있습니다.

## 문서 구성

| 순서 | 파일 | 용도 |
|------|------|------|
| 1 | [01_시스템_설계서.md](01_시스템_설계서.md) | 전체 아키텍처, DB 설계, 기술 스택, 디렉토리 구조 |
| 2 | [02_상세기능_설계서.md](02_상세기능_설계서.md) | API 스펙, 14개 에이전트 상세, 클라이언트 구현, 날짜 파서 |
| 3 | [03_구현_가이드.md](03_구현_가이드.md) | Phase별 구현 순서, Claude에게 줄 지시문, 주의사항 |
| 4 | [04_CLAUDE_MD_템플릿.md](04_CLAUDE_MD_템플릿.md) | 새 프로젝트용 CLAUDE.md 템플릿 |
| 5 | [05_프롬프트_레시피.md](05_프롬프트_레시피.md) | Claude Code에 복붙할 프롬프트 8개 + 디버깅 프롬프트 |
| 6 | [06_고급_청킹_설계서.md](06_고급_청킹_설계서.md) | 5가지 청킹 전략 비교, chunker_v2 설계, 교체 가이드 |

## 사용 방법

### 새 프로젝트에서 재구축할 때

1. `data/sample_paper.csv` 준비
2. `04_CLAUDE_MD_템플릿.md`을 기반으로 `CLAUDE.md` 작성
3. `05_프롬프트_레시피.md`의 프롬프트를 순서대로 Claude Code에 입력
4. 문제 발생 시 디버깅 프롬프트 참조

### 유사 프로젝트를 새로 만들 때

1. `01_시스템_설계서.md`로 아키텍처 이해
2. `02_상세기능_설계서.md`에서 필요한 기능 선택/수정
3. `04_CLAUDE_MD_템플릿.md`의 `{변수}` 부분을 프로젝트에 맞게 수정
4. `03_구현_가이드.md`의 Phase 순서대로 구현

### 읽는 순서

- **빠른 파악**: 01 → 03 (아키텍처 + 구현 순서)
- **상세 구현**: 01 → 02 → 03 (전체 이해)
- **바로 시작**: 04 → 05 (CLAUDE.md + 프롬프트 복붙)
